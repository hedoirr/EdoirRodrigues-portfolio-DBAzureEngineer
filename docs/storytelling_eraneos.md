# Eraneos Storytelling

## 30-second version
I built a production-oriented Azure data platform starting from infrastructure, deployment validation and environment stability. From there, I structured the solution to evolve into end-to-end data engineering with Data Factory, Databricks, SQL serving and business-ready dashboards.

## 60-second version
What I focused on first was making the platform stable. That meant deployments, Databricks workspace provisioning, SQL configuration and storage setup. Once the environment foundation was in place, I designed the next layers the way I would do it in a real project: orchestration with Azure Data Factory, medallion transformations in Databricks, a dimensional serving model in Azure SQL, and a reporting layer for Power BI. I also included quality checks, monitoring and incremental logic because for me a pipeline is not production-ready if it only runs once. It needs to be scalable, observable and reliable.

## What this proves
- I can work beyond theory
- I understand deployment-phase execution
- I think in end-to-end platform terms
- I know the difference between setup and production readiness
